# Governance

PinTwist is a maintainer-led open-source project.

## Maintainers

The project is maintained by iScaleLabs. Maintainers decide what is merged,
released, documented, and published.

## Decision Making

Maintainers may accept, request changes to, or reject contributions based on:

- product direction;
- user experience quality;
- maintainability;
- security and privacy risk;
- license and attribution obligations;
- compatibility with the public-repo safety rules;
- the project's local-only, no-backend constraint.

Open source does not mean every proposed change will be merged. Maintainers are
not obligated to accept changes that are off-direction, too broad, unsafe, or not
worth the long-term maintenance burden.

## Pull Requests

Small fixes may be reviewed directly as pull requests.

Large changes should begin as issues or discussions before implementation,
especially changes to the extension's permissions, data handling, public message
APIs, build pipeline, licensing, branding, or contribution policy.

Maintainers may squash, edit, or reword commits before merge.

## Releases And Distribution

The public repository is the public source lane. Building, signing, and
publishing the extension (for example to the Chrome Web Store) are owned by
iScaleLabs and may use private accounts.

Publishing source code does not grant contributors release or store-publishing
access.

## Security And Conduct

Security reports are handled through `SECURITY.md`.

Community conduct is governed by `CODE_OF_CONDUCT.md`.
